## 🔧 5. 初始化脚本层（第 144-150 行）

```dockerfile
COPY ./init.sh /usr/local/bin/init.sh
RUN sed -i 's/\r$//' /usr/local/bin/init.sh && \
    chmod +x /usr/local/bin/init.sh
```

**init.sh 的功能**：
1. ✅ 检查和修复挂载目录权限
2. ✅ 创建基础配置文件
3. ✅ 根据环境变量同步配置（AI 模型、IM 渠道等）
4. ✅ 启动 OpenClaw Gateway 服务

## 🚀 8. 启动入口层（第 175-183 行）

```dockerfile
WORKDIR /home/node
ENTRYPOINT ["/bin/bash", "/usr/local/bin/init.sh"]
```

### 容器启动流程

```
1. 执行 /usr/local/bin/init.sh
   ↓
2. 检查并修复挂载目录权限
   ↓
3. 如果 openclaw.json 不存在，创建基础配置
   ↓
4. 根据环境变量同步配置到 openclaw.json
   - AI 模型配置（MODEL_ID, BASE_URL, API_KEY 等）
   - IM 渠道配置（飞书、钉钉、QQ、企业微信）
   - Gateway 配置（端口、Token、绑定地址）
   ↓
5. 打印运行时摘要
   ↓
6. 以 node 用户身份启动 OpenClaw Gateway
   ↓
7. 等待 Gateway 进程退出
```

## 💡 实际运行示例

### 启动容器

```bash
docker run -d \
  --name openclaw-gateway \
  -e MODEL_ID=gemini-3-flash-preview \
  -e BASE_URL=http://localhost:3000/v1 \
  -e API_KEY=your-api-key \
  -e FEISHU_APP_ID=cli_xxx \
  -e FEISHU_APP_SECRET=xxx \
  -v ~/.openclaw:/home/node/.openclaw \
  -p 18789:18789 \
  -p 18790:18790 \
  justlikemaki/openclaw-docker-cn-im:latest
```

### 容器内部实际运行的命令

```bash
node openclaw gateway run \
  --bind lan \
  --port 18789 \
  --token 123456 \
  --verbose
```
